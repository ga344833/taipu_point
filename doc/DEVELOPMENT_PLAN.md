## 開發注意事項

### 資料庫設計
- 確保 User、Product、Point、Order 等模型之間的關聯正確
- 考慮未來擴充性，避免過度設計

### 安全性考量
- 所有 API 都需要 JWT 驗證
- 敏感操作（如扣點）需要額外的權限檢查
- 避免 SQL Injection、XSS 等安全漏洞

### 效能考量
- 資料庫查詢優化（使用 `select_related`、`prefetch_related`）
- 避免 N+1 查詢問題
- 考慮快取機制（如需要）

### 程式碼品質
- 遵循 DRY（Don't Repeat Yourself）原則
- 適當的函數與類別命名
- 添加必要的註解與文件字串
